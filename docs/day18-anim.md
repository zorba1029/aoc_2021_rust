# Day 18: Snailfish — 축약 시각화 (Reduction Visualization)

Day 18의 snailfish(달팽이고기) 수는 `[[1,2],[3,4]]`처럼 **중첩된 쌍**이다. 두 수를
더하면 두 쌍을 `[좌, 우]`로 이어 붙인 뒤, 더 이상 줄일 게 없을 때까지 **축약(reduce)**
한다. 이 문서는 그 축약 과정 — explode와 split의 연쇄 — 을 터미널에서 실시간으로
**깊이별 색과 섬광으로 보는** 애니메이션을 추가한 기록이다.

축약 규칙은 단 두 개고, 항상 explode가 split보다 우선한다.

1. **explode**: 네 쌍 안에 중첩된(= 깊이 5) 가장 왼쪽 쌍이 터진다. 왼쪽 값은 왼쪽
   가장 가까운 정규 숫자에, 오른쪽 값은 오른쪽 가장 가까운 정규 숫자에 더해지고,
   그 쌍 자체는 `0`이 된다.
2. **split**: 10 이상인 가장 왼쪽 정규 숫자가 `[v/2, (v+1)/2]` 쌍으로 쪼개진다.

---

## 왜 이게 볼 만한가 — Day 11과 같은 "연쇄"의 구조

Day 11의 문어 격자에서 한 칸의 발화가 이웃을 발화시키는 **사태(avalanche)**를 봤다면,
Day 18은 그 사태가 **격자가 아니라 트리 위에서** 일어난다.

- explode가 옆 숫자에 값을 밀어넣는다 → 그 숫자가 10을 넘는다 → split으로 쪼개진다
  → 깊이 5의 새 쌍이 생긴다 → 또 explode한다...

규칙 어디에도 "연쇄를 일으켜라"는 말이 없는데, 한 번의 더하기가 수십 번의
explode/split을 줄줄이 촉발한다 (sample-1의 첫 더하기만 해도 **21번**). 부분 규칙을
들여다봐도 전체 거동을 예측할 수 없는, Day 11과 똑같은 **창발(emergence)**이다.

차이는 **자료구조의 평탄화**에 있다. 풀이 본체(`tree_handler.rs`)는 `Rc<RefCell>`
트리에 부모 포인터(`Weak`)를 달아 좌/우 이웃을 타고 올라가며 찾지만, 애니메이션은
같은 수를 **납작한 토큰 리스트**(`[` `]` `,` 숫자)로 다룬다. 이러면
"왼쪽 이웃 = 왼쪽으로 스캔하다 처음 만나는 숫자"가 되어 빌림(borrow) 씨름 없이
한 줄로 끝나고, 브래킷이 그대로 남아 있어 **글자마다 중첩 깊이로 색칠**할 수 있다.

## 만든 것

| 파일 | 역할 |
|---|---|
| `src/advent/day18/animate.rs` | 애니메이션 본체 (기존 풀이는 손대지 않음) |
| `src/bin/day18_anim.rs` | 실행 진입점 — **flat 색상 텍스트 뷰** |
| `src/bin/day18_tree_anim.rs` | 실행 진입점 — **flat + 트리 합본 뷰** |
| `src/advent/day18/mod.rs` | `pub mod animate;` 한 줄 추가 |

세 뷰(flat / tree / 합본)는 **같은 축약 엔진**(토큰 기반 explode/split)을 공유한다.
`animate_with()`가 프레임마다 렌더 함수를 호출하는 구조라, 뷰들은 *그리는 방식만*
다르다. 공통 본문은 헬퍼로 분리돼 있다 — `flat_body`(색칠된 한 줄 문자열),
`tree_body`(들여쓰기 `├─`/`└─` 트리), `color`(강조/깊이 색), `start_frame`/`end_frame`.
`render`=flat, `render_tree`=tree, `render_both`=둘 다(`day18_tree_anim`이 사용). 트리는
매 프레임 토큰 리스트를 이진 트리로 파싱해 그리고, 어느 뷰든 같은 highlight 데이터를
써서 explode/split 대상·이웃·결과가 동시에 강조된다.

기존 풀이(`day_18.rs`, `tree_handler.rs`, `tokenizer.rs`)와 `log` 기반 출력은 그대로
두고, **별도 모듈 + 별도 바이너리**로 비침투적으로 추가했다.

## 실행 방법

실제 터미널(VS Code 통합 터미널 OK, truecolor 지원)에서 실행한다.

```bash
# sample-1, 전체 줄 합산 (기본값)
cargo run --release --bin day18_anim

# 다른 샘플
cargo run --release --bin day18_anim -- sample-2

# 진짜 퍼즐 입력, 앞 5줄만 더하기 (입력이 100줄이라 길어서 제한용)
cargo run --release --bin day18_anim -- input 5

# flat + 트리 합본 뷰 (인자는 위와 동일)
cargo run --release --bin day18_tree_anim
cargo run --release --bin day18_tree_anim -- input 3
```

> 합본 뷰는 한 화면에 flat 문자열과 트리를 함께 그린다. 트리는 들여쓰기형이라
> 깊이/너비에 강하지만, 누적 합이 커지면 잎이 수십 개로 늘어 화면 높이를 넘을 수
> 있다. 그럴 땐 `arg2`로 더할 줄 수를 줄이면 된다.

인자:

- `arg1` — 입력 선택: `sample-1` | `sample-2` | `input` (또는 raw 경로). 기본값 `sample-1`
- `arg2` — 합산할 최대 수 개수(첫 수 + 더하기들). 기본값: 전체

> 입력 파일에서 `#`로 시작하는 줄과 빈 줄은 건너뛴다 (`day_18.rs`와 동일).

## 무엇을 보게 되나

- **중첩 깊이 → 색**: 얕은 파랑 → 청록 → 호박색 → 빨강(깊이 5, 곧 explode). 깊을수록
  뜨겁다 = explode가 임박했다는 신호.
- **explode**: 터질 쌍이 밝게 강조되고(노란 섬광), 그 값을 받는 좌/우 이웃 숫자가
  **초록으로 깜빡**인 뒤, 쌍이 `0`(자홍색)으로 주저앉는다.
- **split**: 10 이상 숫자가 강조된 뒤 `[v/2, (v+1)/2]` 쌍(자홍색)으로 갈라진다.

상단 헤더에 `add i/N · op k`와 현재 단계(`explode`·`split`·`settled`)가 떠서,
한 번의 더하기가 몇 번의 축약을 거치는지 한눈에 보인다.

- **2줄 컨텍스트**: 헤더 아래에 더하기 전의 상태가 입력 줄과 같은 형식으로 뜬다.
  `so far :` 는 지금까지 완성된 누적 합, `adding :` 는 이번에 더할 입력 줄. 그래서
  "이 중간결과에 이 줄을 더한다"가 명확히 보인다. (첫 수는 `number 1/N:` 한 줄.)
- **reduce 텍스트 줄**: explode/split이 진행되는 동안 제일 윗줄에 `reduce :` 로 현재
  수를 평문 텍스트로 보여주고 매 연산마다 갱신한다. 색칠된 본문(flat·tree)과 별개로,
  축약이 한 단계씩 어떻게 바뀌는지 텍스트로 따라 읽을 수 있다.
- **합본 레이아웃**(`day18_tree_anim`): 2줄 컨텍스트 아래에 flat 문자열, `── tree ──`
  구분선, 그리고 같은 수의 트리가 함께 그려진다. 두 표현이 동시에 같은 explode/split을
  강조하므로, "한 줄짜리 표기"와 "트리 구조"가 어떻게 대응하는지 나란히 볼 수 있다.

### 한 연산의 프레임 구조

연산 하나당 **두 프레임**이다 — "직전"(대상 강조)과 "직후"(결과 강조):

```
explode    : 깊이 5 쌍 + 받을 이웃 강조  →  쌍이 0으로
split      : 10 이상 숫자 강조           →  [v/2,(v+1)/2] 쌍으로
settled    : 한 줄이 완전히 축약된 정착 상태
```

## 동작 원리 (구현 메모)

- **토큰 리스트**: `enum Tok { Open, Close, Comma, Num(i64) }`의 `Vec`. 풀이의
  `Rc<RefCell>` 트리 대신 평탄한 형태라, 연산이 단순한 `Vec::splice` 편집이 되어
  프레임 단위로 끊어 보여주기 쉽다.

- **explode 탐색**: `[` 를 만날 때마다 깊이를 세다가 깊이 5이고 뒤가
  `숫자 , 숫자 ]` 패턴이면 그 쌍이다 (snailfish 불변식상 깊이 5의 가장 왼쪽 쌍은
  항상 정규 숫자 둘). 왼/오른 이웃은 그 위치에서 좌/우로 스캔하다 처음 만나는 숫자.

- **split 탐색**: 10 이상인 첫 `Num`. `[v/2, (v+1)/2]`로 `splice`.

- **magnitude**: 스택으로 계산. `]`를 만나면 스택 위 두 값을 꺼내 `3*좌 + 2*우`로
  접는다.

- **ANSI 렌더링** (Day 11과 동일 машинери):
  - 대체 화면 버퍼 진입/복귀: `\x1b[?1049h` / `\x1b[?1049l`
  - 커서 숨김/복원: `\x1b[?25l` / `\x1b[?25h`
  - 제자리 다시 그리기: 매 프레임 `\x1b[H`(홈) + `\x1b[J`(끝까지 지움) — 길이가
    가변이라 줄 단위 `\x1b[K` 대신 화면 끝까지 지운다.
  - 24비트 트루컬러: `\x1b[38;2;{r};{g};{b}m`
  - 긴 누적 수는 `WIDTH`(기본 100열)에서 토큰 단위로 줄바꿈.
